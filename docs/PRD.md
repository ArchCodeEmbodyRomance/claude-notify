# PRD：Claude Code 用户输入提醒

**版本**：1.2
**日期**：2026-04-10

---

## 一、问题陈述

用户在等待 Claude 处理任务时，会切换去做其他事。此时如果 Claude 需要用户输入，用户不知道，导致：
- 以为 Claude 还在跑，其实 Claude 在等他
- 需要批准操作时 Claude 卡住、浪费时间

---

## 二、目标用户

在 IDE 或终端中使用 Claude Code、且习惯多任务切换的开发者。

---

## 三、使用场景

### 场景 A：Claude 需要用户回复
涵盖两种情况：Claude 完成回复等待下一条输入，或 Claude 在执行中途提问。两者从系统角度无区别，统一处理。

**捷径**：点击通知直接激活对话窗口。

---

### 场景 B：Claude 需要权限批准
Claude 要执行某个操作，需要用户授权才能继续。有时效性，Claude 在等待期间阻塞。

**捷径**：直接在通知上点按钮，无需切回窗口。

按钮：
- ✓ 允许（仅此次）
- ✓✓ 本次会话全部允许
- ✗ 拒绝

**超时默认行为**：用户未响应超时后，默认拒绝。

---

## 四、通知内容规范

| 字段 | 内容 |
|------|------|
| 标题 | `[应用名 / 项目名]` + 状态描述 |
| 正文 | Claude 最后一句话的截断摘要 |
| 操作 | 场景 A：点击跳转；场景 B：三个按钮 |

---

## 五、不通知的条件

1. 用户当前父窗口已在前台（检测父进程窗口是否活跃）
2. 场景 B 中该操作已被用户预先设置为自动批准
3. 没有可识别的父 IDE/终端进程

---

## 六、父窗口识别

通过进程树向上遍历（非仅读直接父进程），找到第一个匹配已知 IDE/终端列表的进程。

支持宿主环境：VS Code、Cursor、各 JetBrains IDE、Windows Terminal、iTerm2、Warp 等。

---

## 七、场景 B 审批回路

采用**双通道模型**：

- **通道 A**：hook 立即返回 `permissionDecision: ask`，Claude Code 在终端显示 1/2/3 提示，用户可直接键入
- **通道 B**：同时异步（detached 子进程）发送带按钮的 Toast，用户点击按钮后通过自定义协议模拟对应按键发送至终端

两个通道先到先得，无需去重机制。Toast 的生命周期天然界定了通道 B 的有效窗口：
- Toast 在屏幕上显示时，通道 B 有效
- Toast 消失后（过期或用户关闭）：
  - Windows：操作中心点击仅唤起窗口，不触发审批按钮，通道 B 自动失效
  - macOS：terminal-notifier 返回空值，子进程不发按键，自然退出

PostToolUse hook 负责清理残留子进程（兜底），不承担去重职责。

审批 URL 格式：`claude-focus://key/<1|2|3>/<唯一ID>`，唯一 ID 使用加密随机数生成。

---

## 八、平台支持

| 平台 | 通知方式 | 按钮审批 | 窗口激活 |
|------|---------|---------|---------|
| Windows | WinRT Toast | Toast 按钮 + 自定义协议 + SendInput | SetForegroundWindow（需借前台权限） |
| macOS | terminal-notifier / osascript | terminal-notifier -actions + osascript keystroke | osascript activate |

**macOS 特别说明**：osascript 向其他应用发送按键需要辅助功能权限，首次安装需引导用户在「系统设置 → 隐私与安全性 → 辅助功能」中授权，否则通道 B 失效（通道 A 仍可用）。

---

## 九、已知兼容性风险

| 风险 | 平台 | 应对 |
|------|------|------|
| ConPTY 按键路由 | Windows | 需实测验证 SendInput 能否正确路由至 Claude Code stdin |
| JetBrains 集成终端 | 双平台 | 窗口句柄归属复杂，需实测 |
| JetBrains 进程名为 java | macOS | 需检查命令行参数识别具体 IDE |

---

## 十、非功能要求

- **不阻塞 Claude**：通知发送在 detached 子进程中执行，hook 立即返回
- **零额外强制依赖**：只依赖 Node.js 标准库和系统工具；macOS 的 terminal-notifier 为可选增强
- **唯一 ID 安全性**：审批 URL 中的 ID 使用加密随机数，防止伪造

---

## 十一、使用注意

场景 B 中，若用户已在终端完成审批操作，请勿再点击屏幕上残留的 Toast 按钮，否则会向终端发送多余按键。

---

## 十二、范围外

- 移动端通知
- 通知历史记录
- 与第三方通知服务（Slack、邮件等）集成
